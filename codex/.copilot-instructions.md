# Copilot/Codex Instructions ‚Äî Smart Store Pro

## Project Context

**Smart Store Pro** ‚Äî Gutenberg block plugin for WooCommerce (~50 blocks).

```
blocks/          ‚Üí JS/JSX (editor, save, dynamic CSS, view)
src/             ‚Üí PHP PSR-4 classes (server render, attributes, CSS)
```

**Block Tiers:**
- `element/*` ‚Äî Atomic fields (image, price, rating, add-to-cart)
- `smart/*` ‚Äî Layout primitives (container, heading, button)
- `showcase/*` ‚Äî Composites (product-grid, carousel, smart-cart)
- `builder/*` ‚Äî Page components (advanced-filters, breadcrumb)

```bash
npm run start          # Dev mode
npm run build          # Production build
npm run build:blocks   # Editor bundle only
npm run build:blocks-php # Regenerate src/Blocks/blocks.php from block.json
```

## Coding Rules

**Security (Non-Negotiable):**
- PHP: Every file starts with `defined('ABSPATH') || exit;`
- Sanitize ALL inputs: `sanitize_text_field()`, `absint()`, `wp_kses_post()`
- Escape ALL outputs: `esc_html()`, `esc_attr()`, `esc_url()`
- Database: `$wpdb->prepare()` ‚Äî no raw SQL
- Auth: `current_user_can()` + `wp_verify_nonce()`

**React:**
- Use `@wordpress/*` packages ‚Äî never jQuery
- `useBlockProps()` in edit, `useBlockProps.save()` in save
- No `dangerouslySetInnerHTML` with unsanitized content
- Correct `useSelect` dependency arrays

**i18n:** Always use `'woo-product-slider-pro'` text domain

**CSS Parity:** `dynamicCss.js` and `*CssGenerator.php` must produce identical output

## Block Registration Pattern

```jsx
import { registerBlockType } from '@wordpress/blocks';
import Edit from './edit';
import Save from './save';
import metadata from './block.json';

registerBlockType('sp-smart-store/block-name', {
    title: __('Block Title', 'woo-product-slider-pro'),
    category: 'smart-store-blocks',
    icon,
    attributes: metadata.attributes,
    usesContext: ['postId', 'query'],
    edit: Edit,
    save: Save,
    supports: { html: false, align: true },
});
```

## Adding a Block

1. Create `blocks/blocks/<tier>/<name>/block.json`
2. Add to `BLOCK_MAPPING` in `scripts/generate-blocks-php.js`
3. Run `npm run build:blocks-php`
4. Create JS: `index.jsx`, `edit.jsx`, `save.jsx`, `dynamicCss.js`
5. Import in `blocks/blocks/index.js`
6. Create PHP: `src/Blocks/<Tier>/<Name>/Block.php`, `*Attributes.php`, `*CssGenerator.php`
7. Register in `src/Includes/BlockManager.php`

## Common Issues

| Issue | Fix |
|-------|------|
| Block not registering | Add to BLOCK_MAPPING, run `npm run build:blocks-php` |
| Module not found | Add import to `blocks/blocks/index.js` |
| CSS not applying | Check uniqueId scoping, verify dynamicCss.js matches CssGenerator.php |
| REST API 404 | Check namespace is `wpspro/v1`, flush permalinks |

## Planning Protocol

For new blocks (3+ files):
1. Read `../common/patterns.md` and `../common/conventions.md`
2. Study 2 existing blocks of the same tier
3. Create plan: files, attributes, context, data fetching
4. Wait for user approval before coding

**Golden Rule:** Plan first for anything touching 3+ files.
