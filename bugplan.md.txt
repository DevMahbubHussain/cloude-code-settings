Why ID-Based Prompting is the Best Approach:

Prevents "Spaghetti Fixes": If you just say "Fix Phase 1 Layout," the AI might rewrite your entire CSS file to fix one margin issue. Telling it "Fix SC-001" forces it to only touch line 161-188 of that specific file.

No Context Loss: As the chat gets longer, AI gets confused. Referencing the ID makes the AI re-read the specific block in the markdown file, refreshing its memory on exactly what the bug is.

Easy Rollback: If the AI messes up SC-004, you know exactly which code to revert in Git. If it messed up "Phase 2 Security," you have no idea what it changed.

Clear Progress Tracking: Your BUGFIX_PLAN.md becomes a real-time checklist. At any point, you can search the file for [FIXED] vs unfixed to see exactly what's left.
